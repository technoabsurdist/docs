## Quick Start for Contributors

<Steps>
<Step title="Fork and Clone">
Fork the repository and clone it locally:

```bash
git clone https://github.com/YOUR_USERNAME/chisel.git
cd chisel
```
</Step>

<Step title="Setup Environment">
Create and activate a virtual environment:

```bash
python -m venv .venv
source .venv/bin/activate  # Linux/Mac
# OR .venv\Scripts\activate  # Windows
```
</Step>

<Step title="Install in Development Mode">
Install Chisel CLI in development mode with all dependencies:

```bash
pip install -e .[dev]
```
</Step>

<Step title="Verify Setup">
Run tests to ensure everything works:

```bash
ruff check src/ examples/
pytest tests/
```
</Step>
</Steps>

## Code Standards

### Linting and Formatting

We use **Ruff** for both linting and formatting to ensure consistent code quality:

<CodeGroup>
```bash Check Code Style
# Check for style and quality issues
ruff check src/ examples/ tests/

# Auto-fix issues where possible
ruff check src/ examples/ tests/ --fix

# Format code
ruff format src/ examples/ tests/

# Check if formatting is needed
ruff format src/ examples/ tests/ --check
```

```bash Run All Quality Checks
# Complete quality check script
#!/bin/bash
echo "🔍 Running code quality checks..."

echo "📏 Checking code style..."
ruff check src/ examples/ tests/

echo "✨ Checking formatting..."  
ruff format src/ examples/ tests/ --check

echo "🧪 Running tests..."
pytest tests/

echo "✅ All checks passed!"
```

```toml Ruff Configuration
# pyproject.toml configuration
[tool.ruff]
line-length = 100
target-version = "py38"

[tool.ruff.lint]
select = [
    "E",  # pycodestyle errors
    "W",  # pycodestyle warnings
    "F",  # pyflakes
    "I",  # isort
    "B",  # flake8-bugbear
    "C4", # flake8-comprehensions
]
ignore = [
    "E501",  # line too long (handled by formatter)
]

[tool.ruff.format]
quote-style = "double"
indent-style = "space"
```
</CodeGroup>

```python
from typing import Optional

logger = logging.getLogger(__name__)


class ChiselAuthenticationError(RuntimeError):
    """Raised when authentication with Chisel backend fails."""
    pass


def load_credentials() -> Optional[dict]:
    """Load stored credentials from file.
    
    Returns:
        Credentials dictionary or None if not found/invalid.
        
    Raises:
        ChiselAuthenticationError: If credentials are corrupted.
    """
    try:
        with open(credentials_path) as f:
            credentials = json.load(f)

            raise ChiselAuthenticationError(
                "Stored credentials are invalid. Please re-authenticate."
            )
            
        return credentials
        
    except FileNotFoundError:
        logger.debug("No credentials file found")
        return None
        
    except json.JSONDecodeError as e:
        raise ChiselAuthenticationError(
            f"Credentials file is corrupted: {e}. Please re-authenticate."
        ) from e
```

```python
import pytest

from unittest.mock import patch, MagicMock

from chisel import ChiselApp, GPUType
from chisel.auth import authenticate, clear_credentials


class TestChiselApp:
    """Test suite for ChiselApp functionality."""
    
    def test_app_creation_with_gpu_type(self):
        """Test ChiselApp creation with GPU type."""
        app = ChiselApp("test-app", gpu=GPUType.A100_80GB_2)
        
        assert app.name == "test-app"
        assert app.gpu == "A100-80GB:2"
        
    def test_app_creation_defaults(self):
        """Test ChiselApp creation with default values."""
        app = ChiselApp("test-app")

        assert app.gpu is None
        assert app.upload_dir == "."
        
    @patch.dict(os.environ, {'CHISEL_ACTIVATED': '1'})

        """Test ChiselApp activation with environment variable."""
        app = ChiselApp("test-app")
        
        assert app.activated is True

        mock_post.return_value.json.return_value = {
            'job_id': 'test-123',
            'status': 'success'
        }
        
        app = ChiselApp("test-app", gpu=GPUType.A100_80GB_1)
        
        # Test functionality that submits jobs
        # ...
        
        # Verify backend was called correctly
        mock_post.assert_called_once()
```
</Accordion>

<Accordion title="Mocking and Fixtures" icon="wrench">
Use mocking to isolate tests and avoid external dependencies:

```python
import pytest
from unittest.mock import patch, MagicMock, mock_open


@pytest.fixture
def mock_credentials_file():
    """Fixture providing mock credentials file."""
    credentials = {
        "api_key": "test-api-key-123",
        "backend_url": "http://localhost:8000",
        "expires_at": "2024-12-31T23:59:59Z"
    }
    return json.dumps(credentials)


@pytest.fixture
def temp_credentials_dir(tmp_path):
    """Fixture providing temporary credentials directory."""
    creds_dir = tmp_path / ".chisel"
    creds_dir.mkdir()
    return creds_dir


class TestAuthentication:
    """Test authentication functionality."""
    
    def test_load_existing_credentials(self, mock_credentials_file):
        """Test loading valid credentials from file."""
        with patch('builtins.open', mock_open(read_data=mock_credentials_file)):
            with patch('os.path.exists', return_value=True):
                credentials = load_credentials()
                
        assert credentials['api_key'] == 'test-api-key-123'
        
    @patch('webbrowser.open')
    @patch('requests.post')
    def test_browser_authentication(self, mock_post, mock_browser):
        """Test browser-based authentication flow."""
        mock_post.return_value.json.return_value = {'api_key': 'new-key-456'}
        
        result = authenticate()
        
        mock_browser.assert_called_once()
        assert result == 'new-key-456'
```
</Accordion>

**Key Methods**:

```python
class ChiselApp:

    def capture_trace(self, trace_name: str = None, **kwargs) -> Callable
    def _submit_job(self, script_path: str, args: List[str]) -> str
    def _package_code(self) -> bytes
```

**Key Responsibilities**:
- Browser-based authentication flow

- Automatic token refresh

**Key Functions**:

</Tab>

<Tab title="CLI Entry Point (__init__.py)">

**Key Responsibilities**:

**Execution Flow**:

### Execution Modes

<AccordionGroup>
<Accordion title="Local Mode (CHISEL_ACTIVATED != '1')" icon="desktop">
**When**: Running without `chisel` command

**Behavior**:
- ChiselApp runs in inactive mode
- Decorators are pass-through (no-op)
- Code executes normally on local machine
- No authentication or upload required

```python
# Local execution
python my_script.py
# → Decorators do nothing
# → Runs on local CPU/GPU
```
</Accordion>

<Accordion title="Chisel Mode (chisel command)" icon="cloud">
**When**: Using `chisel` command

**Behavior**:
- Sets `CHISEL_ACTIVATED=1` environment variable
- Activates GPU functionality in ChiselApp
- Handles authentication automatically  
- Uploads code and runs on cloud GPU

```python
# Cloud execution
chisel python my_script.py
# → Sets environment variable
# → Uploads and runs on cloud
```
</Accordion>

<Accordion title="Backend Mode (CHISEL_BACKEND_RUN=1)" icon="server">
**When**: Code executing on backend infrastructure

**Behavior**:
- Detects execution on cloud environment
- Skips authentication and upload steps
- Executes functions with GPU tracing enabled
- Streams output back to client

```python
# Backend execution (automatic)
# → Code runs on cloud GPU
# → Output streamed to client
```
</Accordion>
</AccordionGroup>

## Release Process

### Version Management

<AccordionGroup>
<Accordion title="Versioning Strategy" icon="tag">
We follow [Semantic Versioning](https://semver.org/):

- **MAJOR**: Breaking changes (e.g., 1.0.0 → 2.0.0)
- **MINOR**: New features, backwards compatible (e.g., 1.0.0 → 1.1.0)  
- **PATCH**: Bug fixes, backwards compatible (e.g., 1.0.0 → 1.0.1)

**Update locations**:
```bash
# Version must be updated in both places:
# 1. pyproject.toml
version = "0.2.0"

# 2. src/chisel/__init__.py  
__version__ = "0.2.0"
```
</Accordion>

<Accordion title="Release Checklist" icon="list">
Before creating a release:

- [ ] Update version numbers in both files
- [ ] Run complete test suite: `pytest tests/`
- [ ] Test examples manually: `python examples/*.py`
- [ ] Update CHANGELOG.md with new features and fixes
- [ ] Update documentation for any API changes
- [ ] Create git tag: `git tag v0.2.0`
- [ ] Push tag: `git push origin v0.2.0`

**Testing script**:
```bash
#!/bin/bash
echo "🚀 Pre-release testing..."

echo "🔍 Code quality..."
ruff check src/ examples/ tests/
ruff format src/ examples/ tests/ --check

echo "🧪 Test suite..."  
pytest tests/ -v

echo "📝 Example scripts..."
for example in examples/*.py; do
    echo "Testing $example"
    python "$example" || exit 1
done

echo "✅ All checks passed! Ready for release."
```

### PyCharm Configuration

<AccordionGroup>
<Accordion title="Interpreter Setup" icon="gear">
1. Go to **File** → **Settings** → **Project** → **Python Interpreter**
2. Click gear icon → **Add** → **Existing Environment**
3. Select `.venv/bin/python` (or `.venv\Scripts\python.exe` on Windows)
4. Click **OK** to apply

**Verify setup**:
- Check that packages like `ruff` and `pytest` are listed
- Test by running a Python file in the IDE
</Accordion>

<Accordion title="Code Quality Tools" icon="shield-check">
1. Install **Ruff** plugin from marketplace
2. Go to **Settings** → **Tools** → **Ruff**
3. Configure paths:
   - **Ruff executable**: `.venv/bin/ruff`
   - **Configuration file**: `pyproject.toml`
4. Enable **Format on save** and **Lint on save**

**Testing setup**:
1. Go to **Settings** → **Tools** → **Python Integrated Tools**
2. Set **Default test runner** to **pytest**
3. Set **pytest** executable to `.venv/bin/pytest`

### Community Guidelines

<Tip>
Be respectful, collaborative, and helpful when contributing to Chisel CLI.
</Tip>

**Best Practices**:
1. **Start Small**: Begin with documentation improvements or small bug fixes
2. **Ask Questions**: Use GitHub Discussions for questions before starting work  
3. **Follow Standards**: Adhere to code style and testing requirements
4. **Test Thoroughly**: Ensure your changes work correctly and don't break existing functionality
5. **Document Changes**: Update relevant documentation and examples

**Contributing Areas**:
- 🐛 **Bug Fixes**: Fix issues reported by users
- ✨ **Features**: Add new GPU types, authentication methods, or CLI improvements
- 📚 **Documentation**: Improve guides, examples, and API documentation  
- 🧪 **Testing**: Add test coverage and improve test reliability
- 🎨 **UI/UX**: Improve CLI output, error messages, and user experience

Thank you for contributing to Chisel CLI! 🚀
